## Day 2: 데이터 분석 실전

### pandas 2.x

- DataFrame은 2차원 표, Series는 1차원 레이블 배열
- `read_csv`, `read_parquet`로 데이터를 읽고 `shape`, `info`, `describe`, `head`, `sample`로 EDA 수행
- `astype`, `to_datetime`, `loc`로 타입 변환과 조건 선택
- `isna`, `fillna`, `dropna`로 결측치를 확인하고 처리
- 결측 원인을 MCAR, MAR, MNAR 관점에서 판단
- 사분위수와 IQR을 이용해 이상치 후보 탐지
- `groupby().agg()`와 named aggregation으로 그룹별 다중 집계
- `pivot_table`로 교차 집계, `merge`로 열 기반 결합, `join`으로 인덱스 기반 결합
- 체인 할당을 피하고 `.loc`, `.copy()`, `.assign()`을 사용
- `apply()`보다 벡터화 연산과 `.str`, `.dt` 접근자를 우선 사용

### pandas Copy-on-Write 정정 메모

- 교재에는 pandas 2.0부터 Copy-on-Write가 기본이라고 적혀 있으나 공식 문서 기준으로 정확하지 않음
- pandas 2.x에서는 `pd.options.mode.copy_on_write = True`로 선택 활성화
- pandas 3.0부터 Copy-on-Write가 기본이자 유일한 동작
- 이후 예제는 체인 할당을 피하도록 작성해 pandas 2.x와 3.x 전환에 모두 안전하게 구성

### Polars와 DuckDB

- Polars는 Rust·Arrow 기반이며 Eager와 Lazy API를 제공
- Lazy 분석은 `scan_csv → filter → group_by → agg → sort → collect` 흐름
- DuckDB는 CSV·Parquet 파일을 직접 SQL로 조회하고 pandas/Polars로 결과 변환 가능
- Parquet와 Arrow는 열 중심 저장과 데이터 복사 감소에 유리

### 시각화

- Matplotlib: 세밀한 정적 차트와 보고서
- Seaborn: 분포·상관·그룹 비교 등 통계 시각화
- Plotly: HTML로 공유할 수 있는 인터랙티브 차트
- Altair: 선언형 문법을 이용한 빠른 EDA

### 통계와 머신러닝

- 평균·중앙값·분산·표준편차·IQR·왜도·첨도·상관계수 해석
- t-test와 카이제곱 검정의 가설과 p-value 해석
- CRISP-DM의 업무 이해 → 데이터 이해 → 준비 → 모델링 → 평가 → 배포 흐름
- sklearn `Pipeline`과 `ColumnTransformer`로 전처리와 모델 결합
- joblib으로 학습된 전체 파이프라인 저장과 재로딩

### 자동화와 프로젝트 구조

- schedule/cron으로 반복 분석 실행
- Jinja2로 분석 결과를 HTML 리포트에 주입
- 수집, 검증, 변환, 저장 단계를 분리해 테스트 가능한 파이프라인 구성
- 탐색은 노트북, 반복 실행·테스트 대상은 Python 모듈로 분리
- README, 의존성, 테스트, GitHub 이력으로 재현성과 협업성 확보
